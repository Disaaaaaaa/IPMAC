Введение

Компьютерные сети

Компьютерная сеть — совокупность двух и более компьютеров, соединённых каналами передачи информации для совместного использования ресурсов и обмена данными.

✅ Преимущества сети

  • Мгновенная передача файлов всем пользователям
  • Совместный доступ к принтерам и устройствам
  • Общие базы данных и программы
  • Централизованное управление
  • Снижение затрат на оборудование

❌ Недостатки сети

  • Требует оборудования и настройки
  • Риски безопасности (взлом, вирусы)
  • Зависимость от центрального узла
  • Затраты на обслуживание
  • При сбое сети — потеря доступа

Топологии сетей

Шина (Bus)
Все устройства на одном кабеле. Просто, но ненадёжно.
Звезда (Star)
Все подключены к центральному узлу. Надёжно, удобно.
Кольцо (Ring)
Устройства в замкнутой цепи. Равный доступ.
Ячейка (Mesh)
Каждый связан с несколькими. Максимум надёжности.
9.5.1.1

MAC-адрес — физический адрес

MAC-адрес (Media Access Control address) — уникальный идентификатор сетевого устройства, «зашитый» производителем в сетевую карту. Это аппаратный адрес устройства.

Структура MAC-адреса

00: 1A: 2B: 3C: 4D: 5E
OUI — код производителя (пердые 24 бит)
NIC — номер карты (последние 24 бит)
48

16

281

L2

🔑 Ключевые свойства

  • Уникальность — нет двух одинаковых MAC-адресов в мире
  • Аппаратный — прошит в ПЗУ сетевой карты
  • Локальный — действует только в пределах LAN
  • Канальный уровень — работает на Layer 2 модели OSI
  • Формат — 6 байт в шестнадцатеричном виде

💻 Как узнать MAC-адрес

Windows
ipconfig /all
→ «Физический адрес»
Linux / macOS
ip addr show
ifconfig
→ поле «ether»

🔗 Протокол ARP

ARP (Address Resolution Protocol) — связывает IP-адрес с MAC-адресом. Когда компьютер А хочет отправить данные компьютеру Б, он знает только IP-адрес Б. ARP рассылает broadcast-запрос: «Кто имеет IP 192.168.1.5?» Компьютер Б отвечает своим MAC-адресом. Далее данные доставляются по MAC.

A → ARP Request (broadcast): «Кто 192.168.1.5?»
B → ARP Reply: «Это я, мой MAC: 00:1A:2B:3C:4D:5E»
A → Отправляет кадр напрямую на MAC B
9.5.1.2

IP-адрес — логический адрес

IP-адрес (Internet Protocol address) — логический адрес, который назначается устройству в сети для его идентификации и обеспечения маршрутизации данных. В отличие от MAC, IP может меняться.

Структура IPv4-адреса

192. 168. 1. 105
■ Адрес сети (Network) ■ Адрес хоста (Host)
4 октета × 8 бит = 32 бита | Диапазон каждого октета: 0–255

IPv4

  • 32 бита (4 байта)
  • Десятичная запись: 192.168.1.1
  • ~4,3 миллиарда адресов
  • Адреса заканчиваются!
Действующий стандарт

IPv6

  • 128 бит (16 байт)
  • Шестнадцатеричная: 2001:0db8::1
  • ~340 ундециллионов адресов
  • Заменит IPv4 полностью
Новый стандарт

Классы IPv4-адресов

Класс Диапазон Применение Частный диапазон
A 1.0.0.0 – 126.x.x.x Крупные сети 10.0.0.0/8
B 128.0.0.0 – 191.x.x.x Средние сети 172.16.0.0/12
C 192.0.0.0 – 223.x.x.x Малые (ЛВС) 192.168.0.0/16
D 224.0.0.0 – 239.x.x.x Multicast —
E 240.0.0.0 – 255.x.x.x Зарезервировано —

🔒 Статический IP

  • Назначается вручную
  • Не меняется при перезагрузке
  • Используется: серверы, принтеры

⚡ Динамический IP (DHCP)

  • Назначается автоматически
  • Может меняться каждый раз
  • Используется: ПК, ноутбуки

💻 Как узнать IP-адрес

Windows
ipconfig
→ «IPv4-адрес»
Linux / macOS
ip addr show
→ поле «inet»

🔌 Сетевая карта (NIC — Network Interface Card)

НазначениеПодключает компьютер к сети (проводной или беспроводной)
АдресСодержит уникальный MAC-адрес, «прошитый» производителем
ТипыПроводная (Ethernet, RJ-45) и беспроводная (Wi-Fi, радиоантенна)
Скорость10 / 100 / 1000 Мбит/с (Ethernet), до 9,6 Гбит/с (Wi-Fi 6)
УстановкаВстроенная (на материнской плате) или внешняя (PCI / USB)
Уровень OSILayer 2 — Канальный
💡 ФактСовременные ПК имеют обе карты: проводную и беспроводную

🔀 Коммутатор (Switch)

НазначениеСоединяет устройства в LAN, передаёт данные только нужному получателю
АдресацияИспользует MAC-адреса (хранит таблицу MAC — порт)
ПередачаUnicast — только на нужный порт, не на все
ПортыОбычно 8, 16, 24, 48 портов
СкоростьПолный дуплекс — не делит полосу между устройствами
Уровень OSILayer 2 — Канальный (некоторые Layer 3)
💡 ФактКоммутатор умнее хаба: «помнит», какое устройство на каком порту

📡 Концентратор (Hub)

НазначениеСоединяет устройства и транслирует данные на ВСЕ порты
АдресацияНе использует адреса — передаёт всем без разбора
ПередачаBroadcast — рассылка на все порты одновременно
ПроблемаКоллизии — все устройства в одном домене коллизий
СкоростьДелит пропускную способность между всеми устройствами
Уровень OSILayer 1 — Физический
⚠️ СтатусУстарел — заменён коммутаторами из-за низкой эффективности

🌐 Маршрутизатор (Router)

НазначениеСоединяет несколько сетей и определяет маршрут для пакетов
АдресацияИспользует IP-адреса и таблицы маршрутизации
ФункцииМаршрутизация, NAT, DHCP-сервер, межсетевой экран
ПрименениеLAN ↔ WAN, LAN ↔ Internet
Уровень OSILayer 3 — Сетевой
💡 ФактДомашний Wi-Fi роутер = маршрутизатор + коммутатор + WAP в одном корпусе
